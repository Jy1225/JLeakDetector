RQ1:工具效果如何?与其他工具相比怎样?
1、用来对比的工具：
与PMD和spotbugs进行比较。

2、衡量当前工具和其他工具效果所用的指标有哪些：
TP_known：命中基准集中已知 bug
FN：基准中的 bug 没被命中
TP_new：不在基准中，但人工确认是真 bug
FP：人工确认是假阳性
precision（P）：(TP_known + TP_new) / (TP_known + TP_new + FP) 报出来的问题里有多少是真的
recall（R）：TP_known / (TP_known + FN) 已知真实 bug 中有多少被找到了
F1-score：2PR / (P + R)	平衡 precision 和 recall

RQ2:时间和token成本是多少？
1、方法：
在测试时进行统计。

我们还需要实现能够在测试时对时间和所花费token进行统计的代码逻辑

2、指标：
（1）不同项目各自的运行时间
（2）不同项目下平均命中每个真实bug的耗时
（3）所有测试项目平均运行时间
（4）所有测试项目平均命中每个真实bug的耗时
（5）不同项目输入token总数
（6）不同项目输出token总数
（7）不同项目总token消耗
（8）不同项目下平均命中每个真实bug需要的token数量
（9）所有测试项目平均总token消耗
（10）所有测试项目平均命中每个真实bug需要的token数量

RQ3:不同大语言模型、不同参数上的表现如何？
调用不同模型的api以及同一模型的不同参数进行测试，比较如下指标：
precision（P）、recall（R）、F1-score、
所有测试项目平均运行时间、所有测试项目平均每个真实bug的耗时、
所有测试项目平均总token消耗、所有测试项目平均每个真实bug需要的token数量

RQ4:不同模块对工具效果的影响如何？

将得到的指标进行统计学上的比较即可，得出一系列结论。
